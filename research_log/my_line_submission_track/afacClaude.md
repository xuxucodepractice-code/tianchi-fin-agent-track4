## 一、这个比赛是什么（一句话版）

天池 AFAC2026 赛题四：做一个基于 **Qwen** 的**金融长文本问答 Agent**，读保险条款/监管法规/金融合同/财报/研报这类长文档，定位证据 → 推理 → 输出单选/多选/判断题答案 → 统计 Token → 生成 `answer.csv` 提交。

- 官方页面：https://tianchi.aliyun.com/competition/entrance/532486
- 答疑钉钉群：174890011923

### 三条硬约束（决定技术路线）
1. **正式推理问答必须用 Qwen 系列 API**，评测基准模型是 `Qwen3.6-plus`；只能通过**阿里云百炼**或**魔搭社区**调用。
2. **禁止 embedding 模型**做检索/rerank/推理 → 只能用关键词/BM25/规则/条款号检索，不能用向量数据库。
3. 预处理阶段（OCR、PDF 转文本）可以用非 Qwen 工具，但其结果不得延伸到正式检索和答题。

### 关键时间（很紧）
- **A 榜：2026-06-08 ~ 2026-07-21 20:00**
- **B 榜：2026-07-22 ~ 2026-07-24 17:00**
- ⚠️ **A 榜必须有有效成绩，才能进 B 榜。** 从 7/1 算，A 榜只剩约 20 天。
- B 榜不给 `doc_ids`，要自己检索，难度更高。

### 评分公式
```
Accuracy = 答对数 / 总数
TokenBudget = 5,000,000
TokenScore = max(0, min(1, (TokenBudget - TotalTokens) / TokenBudget))
FinalScore = 100 × Accuracy × (0.7 + 0.3 × TokenScore)
```
准确率是大头，Token 效率最多影响 30%。多选题无部分分，漏选/错选/多选都算错。

### ✅ 提交格式（官方原文确认 2026-07-04，照此写 answer.csv）
5 列固定：`qid, answer, prompt_tokens, completion_tokens, total_tokens`
- **第一行必须是 summary 行**：answer 留空，后三列填**所有题总和**。评分看的就是这行的 total_tokens。
- 之后每行一道题：qid + 答案 + 该题各自 token。
- 答案格式（错一点就判错）：单选一个大写字母 `A`；判断 `A`/`B`；多选**字母升序无分隔符** `ABC`（不能 `A,B,C`）。

示例：
```
qid,answer,prompt_tokens,completion_tokens,total_tokens
summary,,3627557,629,3628186
ins_a_001,B,37201,1,37202
```

### ✅ 模型名策略（官方原文确认 2026-07-04）
- 红线：正式答题必须用 **Qwen 系列**（不许换别的牌子）。`qwen-plus-latest` 也是 Qwen 系列 → 合规。
- 官方评测基准型号是 **qwen3.6-plus**（那张49%基线就是它跑的）。
- 策略：**本地开发用 `qwen-plus-latest`**（能调通）；**真提交那次优先试 `qwen3.6-plus`**，能通就用它（和评测同款、本地线上一致），不通就用 latest 并注明。代码里模型名是一个变量，提交前一行改掉即可。

### 📌 留意（不影响 A 榜，B 榜前15名才要）
B 榜前 15 名需提交 `evidence.json`（每题答案+证据溯源:哪份文档哪一条）。A 榜只交 answer.csv。设计时留口子，以后不返工。

## 二、被澄清的核心疑问（防止再懵）

### Q1：官方给 baseline 了吗？
**没有可运行代码。** 官方只给了一张分数表（Qwen-plus，A 榜准确率 49%）当"靶子"，没有代码/notebook。网上那个 `li-xiu-qi/afac2026-financial-qa` 是网友写的、且用了 Step 模型（**违规**），只能参考思路不能照抄。
**结论：提交线不是"复现 baseline"，而是从零搭一个最小系统。**

### Q2：做 Agent 和训练模型的区别？
- **训练模型** = 从头造一个"大脑"（改模型内部参数），成本极高，比赛**禁止**。
- **做 Agent** = 拿现成的大脑（Qwen），写它**外面**的工程（检索、prompt、后处理、Token 优化），让它会干一件具体的事。
- 本比赛**只能做 Agent**：大家用同一个 Qwen，比的是**谁的外围工程做得好**。官方 baseline 只有 49%，就是因为把大脑用得太粗（整篇文档硬塞）。

### Q3：数据（public_dataset_a.zip）用来干什么？不是训练？
- **不是训练数据，是 Agent 的输入原料 + 检测标尺。**
- 桌面上的 zip = 官方 A 榜数据，和仓库里已解压的 `public_dataset_upload/` 是同一份 → **用仓库里那份即可，桌面 zip 当备份，别再解压进仓库。**
- 检测分两层：本地拿"人肉解过的题"当标尺粗测（官方没给标准答案）；线上提交 answer.csv 让官方判真分。

### Q4："不懂代码，不还是用 AI 做 AI 吗？"（最关键）
- **"用 AI 写代码"没问题，"用 AI 替你思考"才有问题。** 区别在主导权：
  - ❌ "帮我把 baseline 改到高分" → AI 替你思考，赛后你一头雾水。
  - ✅ "我判断监管题要按条款号检索，你帮我写成代码" → 你主导、AI 是工具，赛后你懂了。
- 比喻：**建筑师不亲手砌砖，但房子是他设计的。** 你不写每行代码，但系统怎么设计、每个模块为什么存在，是你想明白的。
- **不懂代码不是问题，不懂思路才是问题。**

### Q5：学习线要懂的到底是什么？
不是懂代码语法，是懂这 5 个判断（全是思路，不是代码）：
1. 一道题，人是怎么解的？
2. 哪些步骤该交给代码，哪些必须模型判断？
3. 证据怎么找才准（关键词？条款号？）？
4. 模型该被喂什么、被问什么才答得对（prompt）？
5. 一个完整系统为什么分这些模块，缺了会怎样？

### Q6：网上说的"图识别/MCP/多agent routing/monitor agent"要学吗？
**方向对，但对本比赛超纲，别碰。** 那是做通用 Agent 产品的"满汉全席"。本比赛只用得到其中三道菜：
- ✅ 记忆/上下文管理（动态压缩）
- ✅ 工具调用出错优化（检索不全/召回不好）← 正中要害
- ✅ 端到端评测（跑 100 题看分）
- ❌ 图识别、MCP、多 agent routing、monitor agent → 用不到

### Q7：那个 "Agent Arena" 项目有启发吗？
基本没有——它是"评测 Agent 的擂台"，你们是"上场的选手"，方向垂直。唯一可借的一点：**分领域、多版本对比着看错题**，而不是只看一个总分（这本来就在你们 roadmap 阶段 03 里）。

## 三、双线策略（队友已写进仓库）

参见 `tianchi-fin-agent-track4/research_log/2026-06-29_two_track_strategy.md`。

| | 学习理解线（队友） | 提交反馈线（我） |
|---|---|---|
| 位置 | `workspace/01_agent_learning/` | `workspace/03_baseline_improvement/submissions/` |
| 目标 | 慢慢搞懂人怎么解题、工具怎么辅助、baseline 每个模块为什么存在 | 尽快跑通提交、拿真实线上反馈 |
| 节奏 | 慢，重理解、可解释、记录 | 快，重跑通、反馈、失败记录 |

**为什么分开：** 学习线不被提交压力打乱，提交线不被过度研究拖慢。

**怎么交融取得进步（一个循环）：**
```
【提交线】跑通交一版 → 线上分数 + 发现哪类题错得多
     ↓
【学习线】针对错题深挖：为什么错？怎么找证据？→ 产出理解 + 小工具
     ↓
【提交线】把工具/规则装进系统，再交一版 → 分数变化
     ↓（循环）
```
- 提交线负责"发现哪里疼"，学习线负责"搞懂为什么疼、怎么治"。
- 提交线用真实分数**指哪打哪**，学习线**把每一枪打准**。

**现成可复用的交融点（我直接白嫖学习线成果）：**
- 队友的 `inspect_question.py`（读题）、`map_doc_id.py`（doc_id→路径）、`search_keyword.py`（关键词搜索）→ 提交线直接用。
- 队友人肉解的 `ins_a_007` 答案 **BC** → 我本地检测系统对不对的标尺。

## 六、我和 AI（Claude/Ducc）的协作方式 —— 保证"我真的参与、没被 AI 替我思考"

1. **每写一段代码前，先让 AI 讲清"这段是干嘛、为什么需要它"，我听懂了再让它写。** —— 保证我懂设计。
2. **代码由 AI 落地（我不懂语法没关系），但我负责跑、负责看结果、负责判断下一步改哪。** —— 保证判断权在我。
3. **每跑一版，我自己在 `thinking.md` 记录：分数多少、哪类题错、我猜为什么错。** —— 这份记录是我反哺学习线的东西，也是我"真的参与了"的证据。

原则一句话：**用 AI 写代码可以，用 AI 替我思考不行。设计和判断必须是我的。**
