## 3. 材料讲述了什么

### **ReAct (Reasoning and Acting)**

#### 1.**理解核心动机与问题背景 (Why ReAct?)**

- 背景：现有大模型存在一个问题是通常将“思考”和“行动”分开处理，详细的来讲：

  - **仅推理（Reason Only）**：其依赖于思维链（Chain-of-Thought, CoT），模型的内部是一个黑盒子，其依赖内部知识处理问题而不去外界获取信息因此造成了“幻觉”的存在。
  - **仅行动（Act Only）：** 虽然有些模型被用来在交互环境中生成动作，但它们缺乏使用语言进行抽象高层目标规划或维持工作记忆的能力

- Agent形成的本质，来自于人类智能的启发是在通过现有的LLM所存在智慧推理能力，尝试让其和Action交替进行。通俗来讲，以人为例子：我们在做饭的时候，如果酱油不够了，会去有以下几个选择  一个是出门买酱油；另一个是换一个不需要酱油菜；或者是使用醋替代酱油等等选择。在这些选择形成过程中，需要我们去思考，如同现在LLM的存在。而在这些选择最终被决定后，需要我们去行动。所以**ReAct (Reasoning and Acting)** 所探索的问题是，如何让LLM与Action，也就是思考与行动交替进行。

- 核心思想：这就衍生出了一个核心的思想，既然完全封闭仅仅推理会陷入了闭门造城进而形成幻觉，另一方面仅仅去做Action则会陷入了难以完成抽象高层目标规划。因此ReAct尝试去构建一个：协同效应：**“推理”**帮助模型诱导、追踪和更新行动计划并处理异常；而**“行动”**则允许模型与外部环境（如知识库）交互以收集额外信息。通俗来讲则是：一个当闹钟做判官，一个当手去手机东西，只有脑子会胡思乱想，只有手会乱涂乱画，需要交替进行。

  - **ReAct 构建的双向协同效应：** 如你所言，“脑子做判官，手去收集东西，交替进行”。论文将这种协同效应（Synergy）严谨地概括为两个方向的双向奔赴：

    **Reason to act（为了行动而推理）：** 也就是“做判官”。语言推理帮助模型动态地创建、维持和调整高层行动计划，并在遇到异常时（比如抽屉打不开、搜索不到内容）及时做出应对。**Act to reason（为了推理而行动）：** 也就是“收集东西”。通过动作与外部环境（如维基百科、真实的购物网站等）交互，将获取到的最新、最真实的额外信息融入到下一次的推理中。

#### 2. **掌握 ReAct 的核心机制 (How it Works?)**

##### **核心巧思：动作空间的扩充 (Augmented Action Space)**

- ReAct 的核心想法非常简单：**它将智能体的动作空间扩充为了** *A*^=*A*∪*L***，其中** *L* **代表语言空间（Space of language）**

- 传统模型（只有动作空间A）：“没有下文了”的死循环

  在论文中所提到了一个真实的测试，即 “纯行动（Act-Only）” ，当模型被要求寻找一个“胡椒瓶”的时候，他进行了如下的动作并遇到困境：

  1. 它走到水槽旁边，执行了动作：`拿走 水槽上的胡椒瓶`。
  2. 环境立刻给出了无情的真实反馈：`什么也没发生（Nothing happens）`（因为水槽上根本没有胡椒瓶）。
  3. **悲剧发生了：** 因为没有“语言空间”来消化这个失败，模型无法从上下文中“理解”这个反馈的含义。于是它在下一步又执行了：`拿走 水槽上的胡椒瓶`，环境再次反馈`什么也没发生`。
  4. 正如你所说，它**“没有下文了”**，陷入了无限的死循环和幻觉中，因为它缺少“工作记忆”来告诉自己“这里已经找过了，得换个地方”。

- ReAct 模型（加入了语言空间 L）：“自言自语”破局

  而在同样的场景下，ReAct 模型在遇到“抽屉打不开”或“找不到东西”的异常时，会立刻调用语言空间 $L$ 里的动作（Thought）：

  - **动作：** 打开 抽屉 1。
  - **观察（环境反馈）：** 抽屉 1 是关着的。
  - **想法（自言自语，属于空间 L）：** `“我现在找到了第一个钥匙扣。接下来，我需要找到第二个钥匙扣，它更可能出现在梳妆台、垃圾桶或者保险箱里。我先去梳妆台找找看。”`（*此处参考了论文附加实验中类似的调整计划场景*） 或者类似 `这个抽屉打不开/这里没有刀，我需要去下一个抽屉找`。
  - **结果：** 这句自言自语虽然没有物理地推开任何一扇门，但它成功**更新了模型的上下文（Context）**。模型“记住”了当前的进度，并**动态调整了高层行动计划（handle exceptions and adjust action plans）**。

- 理解：

  我们基本上可以判定，**模型真正缺乏的不是“动手（Act）”的能力，而是“对外部世界的受挫反馈进行消化、分析并重新规划”的能力。**“仅行动（Act-Only）”模型的致命弱点在于：**当客观世界的反馈不符合假设时，它们无法进行复杂的上下文推理。**而 ReAct 引入的“想法（Thought）”，恰恰补足了你提到的**“进一步分析进而执行下一步动作的能力”**。 论文明确将“想法”的核心功能之一概括为：**处理异常并调整行动计划（handle exceptions and adjust action plans）**。 当 ReAct 遇到上述“找不到刀”或“抽屉打不开”的客观受挫反馈时，它的“想法”会立刻发挥作用，在上下文中写下类似`“这里找不到，我应该换一个搜索词”`或者`“这招行不通，我接下来去另一个柜子找”`的推理轨迹，从而成功跳出死循环，引导下一步的新动作。
