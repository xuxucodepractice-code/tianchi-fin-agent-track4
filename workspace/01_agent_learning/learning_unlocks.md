# Learning Unlocks

## Purpose

这个文件用于建立“闯关解锁式学习提醒机制”。

核心原则：

```text
比赛实践是主线。
课程和论文是补给站。
只有到达对应阶段，才解锁对应材料。
```

这样做是为了避免两种情况：

```text
只做比赛，不补理论，后面容易卡住。
只看课程论文，不做比赛，学习容易失控。
```

## Reminder Rule

以后每次进入新阶段前，先检查本文件。

如果当前阶段有解锁内容，需要先提醒：

```text
1. 为什么现在看它。
2. 看完要回答什么问题。
3. 怎么应用回当前比赛代码或流程。
```

提醒不是要求立刻完整学完，而是判断是否需要补一小段理论，再回到比赛实践。

## Notes Rule

每次解锁课程或论文后，如果实际阅读了材料，需要在这里记录笔记：

```text
workspace/01_agent_learning/notes/
```

命名方式：按阶段编号，例如 `notes/stage01_react.md`。

笔记不追求长，但必须包含以下四段：

```text
1. 在此阶段我做了什么（工程上）
   -> 指向对应的实践记录文件，例如 manual_solving/ins_a_007.md，
      让读者能从理论笔记跳回当时的实践记录。

2. 为什么需要读 / 学这份材料
   -> 附材料链接。

3. 材料讲述了什么
   -> 一段可独立阅读的摘要，目标是队友不用先读原文也能看懂重点。

4. 自我想法
   -> 必须包含一句明确的「如何应用回比赛」：
      下一步具体应用到哪个文件、哪个函数或哪个 prompt。
```

历史备注：早期版本用的是三问模板（材料解决什么问题 / 学到哪3个概念 / 哪1个概念可应用），
2026-07-08 起统一改为上面的四段结构，已写的笔记不需要倒推重写。
