# 03 Baseline Improvement

目标：具备改进、强化、修改或重构 baseline 的能力，并用实验结果验证改动。

## 应放内容

- 实验计划
- 实验记录
- 错题分析
- 检索策略改动
- prompt 改动
- 文档解析改动
- Token 优化记录
- 提交版本对比
- 复现说明草稿

## 推荐子目录

```text
experiments/       # 每次实验一个独立记录
error_analysis/    # 错题和失败模式复盘
retrieval/         # 检索策略优化
prompts/           # prompt 版本管理
token_budget/      # Token 统计和压缩策略
submissions/       # 提交版本记录，不放最终大文件
```

## 提交反馈线

`submissions/` 是提交反馈线入口。

这条线的目标不是一开始追求高分，而是尽快跑通一次可复现的提交流程：

```text
baseline 或最小系统
-> 生成提交文件
-> 上传
-> 获得线上反馈
-> 记录格式问题、错题类型和失败原因
```

第一次提交尤其要回答：

- 提交文件格式是否正确？
- 线上反馈如何呈现？
- baseline 或最小系统的初始表现如何？
- 哪些失败值得回到学习理解线继续拆解？
- 哪些模块应该进入后续实验？

提交反馈线和学习理解线要保持隔离但最终交融：

```text
学习理解线提供工具和判断方法。
提交反馈线提供真实错误和优化方向。
```

## 每次实验必须回答

- 改了哪个模块？
- 为什么认为它会提升效果？
- 准确率或人工抽样结果如何变化？
- Token 如何变化？
- 哪类题变好了？
- 哪类题变差了？
- 是否值得保留？

## 通关目标

能独立定位 baseline 的问题，并完成一次可复现的优化闭环：

```text
发现问题 -> 提出假设 -> 修改实现 -> 跑评估 -> 记录结果 -> 决定保留或回滚
```
