# Submissions

这个目录是提交反馈线入口。

它用于记录每一次面向比赛平台的提交尝试，但不存放大型生成文件或不可复现的临时文件。

## 目标

第一次提交的目标不是高分，而是跑通闭环：

```text
跑 baseline 或最小系统
-> 生成提交文件
-> 上传平台
-> 获得反馈
-> 记录问题
```

## 每次提交记录应包含

- 日期。
- 使用的代码版本或 commit。
- 使用的数据范围。
- 生成提交文件的命令或流程。
- 是否成功上传。
- 线上反馈或错误信息。
- 格式问题。
- 明显错题或失败类型。
- 下一步要回到学习线还是实验线处理。

## 建议目录

```text
submissions/
  README.md
  first_submission/
    notes.md
```

`notes.md` 记录过程和结论；大型 `answer.csv`、日志、缓存和临时输出不要默认提交。

## 和学习理解线的关系

提交反馈线负责拿真实反馈。

学习理解线负责解释为什么错、如何改。

后续两线交融方式：

```text
提交线发现错题
-> 学习线做人工复盘
-> 抽象成工具、检索规则或 prompt 改进
-> 回到提交线验证
```
