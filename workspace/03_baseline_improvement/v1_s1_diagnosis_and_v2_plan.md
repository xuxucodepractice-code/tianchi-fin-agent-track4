## 一、核心结论：为什么 v1_s1 提不了分

**一句话：你做的是评审排序第三的实验（检索/证据组织），而排序第一、第二的两大失分源（tf、mcq）一行代码都没动；且检索层的改进被下游未改造的判断层吃掉了。分数不动不是玄学，是必然。**

### 1.1 数字链条

| 指标 | v0（63.2607） | v1_s1 | 变化 |
| --- | --- | --- | --- |
| 答案实际变化题数 | — | **8 题**（全部 insurance multi） | 只动了 8/100 |
| tf 答案分布 | 20/20 全 A | **20/20 全 A** | 零变化 |
| mcq 答案分布 | A=11/15 | **A=11/15** | 零变化 |
| ins_a_001（人工确认应为 B） | A（错） | **A（仍错）** | 重跑了但没救回 |
| 总 token | 1,161,593 | 1,280,096 | +118k → 系数 0.930→0.923，**-0.7 分** |
| judgment 分布 | 202/79/79 | 191/85/84 | 边际变化 |

| --- | --- | --- |
| ins_a_007 | AC → **BC** | ✅ 几乎确定改对（学习线人工+模型复核结论就是 BC），+1 题 |

multi 是整题制：改一个选项通常只是"从错到另一种错"（0 分→0 分）。8 题中只有 1 题有真值背书，净期望 ≈ +1 题（+0.93 分），再减去 token 系数损失 -0.7 分，**净效果 ≈ 0～+0.2 分，完全落在"看不出提分"的区间内**。

### 1.3 四个结构性根因

1. **执行偏离了自己定的优先级**。`v1_architecture_review.md` 第 8 节明确排序：①tf 重做（预期 +4~9 题）→ ③证据组织 → ②mcq 比较。实际只做了③，且只对 insurance。检查代码可证：`agent/prompts.py` 至今只有 `build_option_judgment_messages` 一种逐选项 prompt；`agent/normalize_answer.py:96` 的 "tf 无法确定→fallback A" 原样保留。
2. **检索改进无法穿透判断层**。实验三给了更好的证据（邻近合并、实体配额已进 `retrieve.py`），但下游仍是"逐选项独立判断 + insufficient→fallback"。铁证：ins_a_001 四产品公式已被审计报告全部定位（正确答案 B，置信度高），重跑后**依然答 A**——证据再好，逐选项框架对"排序比较题"就是判不出来。
3. **csrc 解析缺陷修了，但没转化成分**。csrc_0009_att1 已恢复正常文本（chunks 已验证），reg_a_006/015/018 检索结果大幅变化（jaccard 0.111/0.158），但三题答案一个没变——其中 reg_a_018 按审计判断本来就该是 A，撞对的题修好链路当然不涨分。这类"链路修复"是必要的，但它的收益要靠真值是 B 的题才能兑现。
4. **提交记录断档，无法归因**。notes.md 停在 v0 冻结；v1_s1 的线上分数、与基准的 diff、变量清单都没记。一次提交混了三个变量（insurance 检索改造 + csrc 解析修复 + seed 重跑），即使分数有小幅波动也读不出是谁贡献的。

## 二、分层体检表：保留 / 提升 / 优化 / 删除

| 层 | 现状 | 结论 |
| --- | --- | --- |
| 文档解析层 | 68 文档全解析，csrc_0009_att1 已修；表格结构丢失仍在但非主要失分源 | **保留**，不动 |
| 检索层（BM25） | v1_s1 已升级：邻近合并、实体配额、字符预算 | **保留**，停止继续调参（无本地标签，调参不可归因） |
| 证据组织层 | 实体分桶已做；doc→产品名/标题标注是否已进 prompt 需确认 | **提升**：确保每条证据头部有『文档：XX（doc_id=N）第 M 页』来源行 |
| 推理 prompt 层 | **最大病灶**。只有逐选项判断一种形态；tf 把"正确/错误"当选项检索（结构性失效）；mcq 逐项判导致 0-support 纯猜 | **重做**：tf 专用单次判定 + mcq 全题比较，这是 v2 的全部重心 |
| 答案合成层 | 三条坏规则：tf 恒 A fallback；mcq 按 evidence_refs 数决胜；mcq 0-support 直接 fallback A | **删除/替换**这三条；multi 排序去重与格式校验保留 |
| 提交流程层 | validate 护栏、SHA256 冻结、缓存复用（机制优秀） | **保留**；补上"每次提交必记分数+diff"的纪律 |

### 明确的删除清单

1. `normalize_answer.py` tf 分支的 "无法确定 → A"（掷硬币规则，且硬币永远同一面）。
2. `normalize_answer.py` mcq 的 "多 support 按 evidence_refs 数量选"（不可靠决胜）。
3. mcq 0-support 直接 fallback 最前选项（改为触发比较调用）。
4. （行为层面）停止：BM25 参数微调、multi 检索继续加料、解析层重构、B 榜检索优化（7-22 前）、任何省 token 的工作。

## 三、v2 优化计划

总原则：**每次提交只验证一个变量；每个实验先用本地真值标定收益上限再动手；改动只重跑受影响题，其余走缓存。**

### Phase 0：标定与记录补齐（半天，零 token）

1. 把 v1_s1 的线上分数补记进 notes.md（如果没提交过 v1_s1，先提交它拿到读数——它就是实验三的对照数据，不拿白不拿）。
2. 建本地真值集（现已迁移至 `evaluation/local_labels.md`），现成可写入的至少 9 题：
   - ins_a_007 = BC、ins_a_001 = B、ins_a_002 = A（学习线+审计已定）
   - fc_a_013 / fin_a_006 / res_a_003 / reg_a_018 / fin_a_013 = A（审计初判，按核对点快速人工确认）
3. 补标定 3 道非低置信 tf 题（fc_a_003、fc_a_006、fc_a_010），回答"tf 全 A 到底丢几题"——这直接决定 Phase 1 的预期收益是 +2 还是 +8。
4. 以后每次提交后 10 分钟内把 分数 / 与 63.2607 差值 / 答案 diff 题数 / token / 变量说明 记入 notes.md。

### Phase 1：tf 判定框架重做（评审实验一，最高优先级）

- **改动**：
  - `prompts.py` 新增 tf 专用 prompt：对题干陈述本身做一次 `true / false / uncertain` 判定，证据用题干检索结果（复合陈述题按事实点分别给证据——审计发现 4/5 的 tf 是"双文档双事实点"结构）。
  - `reason_qwen.py` tf 走单次调用（省一半 token）。
  - `normalize_answer.py` tf 分支：true→A、false→B；uncertain 触发一次重问，仍不确定才 fallback（方向由 Phase 0 标定结果决定，不再硬编码 A）。
- **回归用例**：res_a_003、fin_a_013 判断必须内部自洽（v0 里标签与理由自相矛盾）；Phase 0 标定的 8 道 tf 全对。
- **重跑范围**：仅 tf 20 题，其余 80 题缓存复用。token ≈ 12 万。
- **提交**：单变量提交，分差即 tf 真实收益。
- **预期**：+2 ~ +8 题（Phase 0 标定后收窄）。

### Phase 2：mcq 全题比较 + 确定性计算校验（评审实验二）

- **改动**：
  - `prompts.py` 新增 mcq 比较 prompt：四个选项+各自证据一次给出，要求比较后选唯一最优；逐选项判断降级为参考信号。
  - 计算题：要求模型在 rationale 列算式，代码侧用 Python 复核算术（确定性校验，合规）。
  - `normalize_answer.py` mcq：删除 evidence_refs 决胜和 0-support fallback，多 support / 0-support 一律走比较调用。
- **硬验收**：ins_a_001 管线必须自动得出 B（公式页码审计报告已全部定位，检索层实体分桶也已就位——这题现在是"只差推理层"的完美验收样本）。
- **重跑范围**：仅 mcq 15 题。token ≈ 25 万。
- **提交**：单变量提交。
- **预期**：+3 ~ +5 题。

### Phase 3：multi 收紧（评审实验四，小样本先行，视 Phase 1/2 分差决定做不做）

- 先离线试点：全选 6 题（fc_a_016、fin_a_005、ins_a_005、ins_a_010、ins_a_017、res_a_014）+ 3-support 抽 10 题，用收紧后的逐要素核对 prompt 重判，人工比对新旧判断，**净收益为正才全量**。
- 风险最高的实验（可能把判对的改错），放最后、必须试点。
- 同时处理 v1_s1 遗留：8 题 multi 变化中无标定的 7 题，在试点时一并人工过一遍，决定保留 v1_s1 版还是回滚 v0 版。

### Phase 4（7-14 前后）：B 榜准备

- 给 68 个文档建"文档卡片"（标题、主体、年份、类型词），卡片级 BM25 选档；`filter_chunks_for_question` 已有插槽。A 榜管线稳定前不启动。
