# Submission, Scoring, And Compliance

整理日期：2026-06-23

## 提交文件

线上评测提交 `answer.csv`，需要同时包含每道题答案和本次评测的 Token 消耗统计。

官方示例列：

| qid | answer | prompt_tokens | completion_tokens | total_tokens |
| --- | --- | ---: | ---: | ---: |
| summary |  | 3627557 | 629 | 3628186 |
| ins_a_001 | B | 37201 | 1 | 37202 |
| ins_a_002 | A | 33063 | 1 | 33064 |

## 答案格式

- 单选题：提交一个大写字母，例如 `A`。
- 判断题：提交一个大写字母，例如 `A` 或 `B`。
- 多选题：提交多个大写字母，按字母顺序排列，不使用空格、逗号或其他分隔符，例如 `ABC`。
- 答案为空、包含非法字符、顺序不规范或与标准答案不完全一致，该题计为错误。

## 评分公式

本赛题采用准确率为主、Token 效率为辅的实时综合评分方式。

```text
Accuracy = Correct / Total
TokenBudget = 5,000,000
TokenScore = max(0, min(1, (TokenBudget - TotalTokens) / TokenBudget))
FinalScore = 100 * Accuracy * (0.7 + 0.3 * TokenScore)
```

含义：

- 准确率决定主体得分。
- Token 效率最多影响 30% 的加权系数。
- 若未提交有效 Token 统计，或 `TotalTokens <= 0`，则 `TokenScore = 0`。
- 最终分数不低于 0。

同分排序：

1. 准确率更高者排名靠前。
2. 若准确率仍相同，`TotalTokens` 更低者排名靠前。
3. 若仍相同，提交时间更早者排名靠前。

## Token 统计范围

Token 统计覆盖回答评测问题全过程中的所有模型 API 调用，包括：

- 检索摘要
- 上下文压缩
- 证据判断
- 答案生成
- 自检
- 其他围绕正式作答产生的 Qwen API 调用

## 模型使用限制

### 允许

- 数据预处理和文档解析阶段可以使用 OCR、版面分析、表格恢复、阅读顺序还原、PDF 转结构化文本等工具。
- 类似 MinerU 的方案可以用于将文档解析为更可读、更结构化的输入。
- 推理问答阶段必须使用 Qwen 系列模型。
- 参赛者仅限通过阿里云百炼平台或魔搭社区调用 Qwen 系列模型 API 完成推理。

### 禁止

- 不得使用任何其他开源或闭源模型替代 Qwen 系列模型完成正式推理问答。
- 不得使用任何 embedding 模型进行检索和推理。
- 不得使用非 Qwen 模型生成的向量、排序结果或召回结果参与正式答题。
- 不得使用非 Qwen 模型参与 rerank、候选过滤、答案投票或纠错。
- 不得直接使用预处理阶段生成的语义摘要、FAQ、结论提炼、知识库等结果参与正式答题。

## 代码审核要求

B 榜成绩排名前 15 名的参赛者需要提交完整代码和复现文档。官方示例结构：

```text
submission.zip
├── answer.csv
├── evidence.json
├── processed_data/
├── agent/
├── script/
├── logs/
├── requirements.txt
└── README.md
```

要求：

- 提交内容应支持按照说明文档一键运行。
- 一键运行后应生成符合要求的结果文件。
- 代码压缩包总大小不超过 1GB。

## Baseline 参考

官方给出的 Qwen-plus 基线：

| 组别 | 正确/样本 | 准确率 | Token 消耗 |
| --- | --- | ---: | ---: |
| A 组 | 49/100 | 49.0% | 2,991,883 |
| B 组 | 13/100 | 13.0% | 3,884,045 |
| 总计 | 30/200 | 15.0% | 6,875,928 |

重要启发：直接将长文档输入通用模型并不能稳定解决任务。需要系统性优化文档解析、规则检索、证据聚合、答案约束和 Token 成本。
