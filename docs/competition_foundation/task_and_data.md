# Task And Data

整理日期：2026-06-23

## 命题背景

金融行业存在大量专业性强、篇幅长、结构复杂的文本资料，例如保险条款、监管法规、债券募集说明书、上市公司年报和行业研究报告。这类文档通常包含期限、比例、例外条件、权利义务关系和跨章节引用。

本赛题要求参赛者构建金融长文本问答 Agent，在不修改基座模型参数的前提下，通过 Qwen 系列模型 API 完成：

- 多文档阅读
- 证据检索与定位
- 跨文本比较
- 复杂条件推理
- 答案规范化
- 动态记忆压缩
- Token 成本控制

## 任务目标

参赛者需要根据出题方提供的金融长文本材料和配套问题集，设计并实现 Agent 的上下文管理、信息压缩、证据检索和问答策略，使系统在尽可能少的 Token 消耗下准确回答问题。

正式线上评测以 `answer.csv` 形式提交。参赛者需要生成答案文件，并附带 Token 消耗统计。

## 题目类型

| 类型 | 说明 | 答案格式 |
| --- | --- | --- |
| 单选题 | 从 A、B、C、D 中选择唯一正确答案 | 单个大写字母，例如 `A` |
| 多选题 | 从 A、B、C、D 中选择所有正确答案 | 多个大写字母，排序后无分隔符，例如 `ABC` |
| 判断题 | 使用 A/B 表示正确或错误 | 单个大写字母，例如 `A` 或 `B` |

## 题目字段

题目 JSON 单条数据包含：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `qid` | String | 题目唯一编号 |
| `domain` | String | 所属领域 |
| `split` | String | 所属组别，取值为 `A` 或 `B` |
| `question` | String | 题干 |
| `options` | Object | A/B/C/D 四个候选选项 |
| `answer_format` | String | 题型，包含 `mcq`、`multi`、`tf` |
| `type` | String | 题目能力标签或业务类型 |
| `doc_ids` | Array[String] | 参考文档编号，A 组提供，B 组不提供 |

## 官方数据领域

| 领域标识 | 领域名称 | 文档数量 | A 榜题数 | B 榜题数 | 主要考查点 |
| --- | --- | ---: | ---: | ---: | --- |
| `insurance` | 保险条款 | 16 | 20 | 20 | 保险责任、身故保险金、退保金额、领取规则、条款比较 |
| `regulatory` | 监管法规 | 26 | 20 | 20 | 法规适用、合规义务、时限要求、监管处罚、条文优先级 |
| `financial_contracts` | 金融合同 | 14 | 20 | 20 | 债券条款、发行信息、评级信息、权利义务关系 |
| `financial_reports` | 财务报表 | 10 | 20 | 20 | 年报指标、经营表现、现金流、研发投入、分红政策 |
| `research` | 行业研报 | 20 | 20 | 20 | 行业趋势、公司比较、指标解读、研究结论核验 |

全量基准题共 200 道，其中 A 榜 100 道，B 榜 100 道。

## 官方推荐解题流程

1. 文档预处理：抽取 PDF 文本，保留标题、页码、章节、表格和条款编号。
2. 文档组织与索引：按领域构建关键词索引和结构化字段索引。
3. 题目解析：识别题型、关键词、实体、时间、金额、指标和候选文档。
4. 证据检索：根据题目和选项分别检索证据，必要时进行多轮查询。
5. 推理作答：让模型基于证据逐项判断选项真假。
6. 答案校验：根据题型约束输出合法答案字母。
7. 结果汇总：生成符合提交格式的 `answer.csv` 或 JSON 文件。

## 初步工程启发

- A 榜应充分利用 `doc_ids`，优先建立 doc_id 到原文、页码、章节、表格的可追溯索引。
- B 榜需要在无 `doc_ids` 情况下完成候选文档检索，因此 A 榜阶段就要验证领域内召回策略。
- 检索不能依赖 embedding 模型，因此应重点考虑关键词、BM25、规则索引、章节编号、实体、金额、期限、表格字段和 Qwen 辅助的查询改写。
- 每次模型调用都要记录 prompt、completion、total token，最后汇总进 `answer.csv`。
